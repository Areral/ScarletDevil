# Scarlet Devil — Site Upgrade Design

**Date:** 2026-06-06
**Status:** Approved (pending spec review)
**Topic:** Visual & functional upgrade of the public dashboard, driven by the existing Scarlet Devil Mansion / Touhou aesthetic.

---

## 1. Context

The project publishes a single-page dashboard (`config/web/template.html` + `style.css` + `main.js`, inlined into `index.html` by `merge.py::build_html`) for an auto-aggregated, multi-stage-verified VPN-subscription service. The page is pure vanilla (no build step); the only external assets are Google Fonts and Font Awesome **Free**.

A separate fix (already applied, out of scope here) repaired the broken deploy: an illegal `"//"` key in `vercel.json` and a stale placeholder `index.html`. A server-side placeholder safety-net and a client-side `healPlaceholders()` self-heal are now in place and must be **preserved** by this work.

## 2. Goals

Upgrade three focus areas, amplifying the existing identity rather than replacing it:

1. **Hero / first impression** — "Live Command Center" (direction A): the hero reads as a living status console.
2. **Statistics showcase** — turn the `#stats` section into a vitrine with animated, data-rich charts.
3. **Subscriptions / onboarding UX** — conservative polish.

Two cross-cutting refinements requested by the user:

- **Cohesive motion** — all animations share one rhythm (a single motion system + one choreography per reveal), never a grab-bag of independent effects.
- **Subtle Touhou lore** — present only where it does not interfere or pull focus (typographic spell-card touches, the existing "Scarlet Mist Barrier" framing). No new loud canvas effects in the main flow.

## 3. Constraints & Decisions (from brainstorming)

| Decision | Choice |
|---|---|
| History storage | **Yes** — `history.json`, rotation to **90 snapshots**, stored in git |
| Tech stack | **100% vanilla** — inline SVG + Canvas + CSS; no new external libraries |
| Data delivery to page | **Embedded** in the page (last ~30 points as a `data-*` attr), not fetched at runtime |
| Icons | Font Awesome **Free only** (Pro icons render blank) |
| Effects layer | Keep `body.fx-extreme` on; do not weaken the existing fun-panel effects |
| Build | No build step; `index.html` regenerated by `build_html` |

## 4. The Motion System ("Алый ритм") — cross-cutting

A single, shared motion language so every animation feels "of one piece".

- **CSS custom properties** (added to `:root`):
  - `--ease-scarlet: cubic-bezier(0.2, 0.8, 0.2, 1)` (unify with existing `--trans`)
  - `--motion-fast: 0.2s`, `--motion-base: 0.4s`, `--motion-slow: 0.8s`
  - `--stagger-step: 60ms`
- **One choreography per reveal.** When a section (hero, stats) enters via the existing `.reveal` IntersectionObserver, a single conductor sequences its children on one timeline: badge → number count-up → sparkline draw → trend chips → bar grows. Children do **not** each start independent timers.
- **Shared primitives**, all using `--ease-scarlet`:
  - `animateCount(el, target, {duration, decimals})` — eased numeric count-up.
  - `drawSparkline(svg, series, opts)` — SVG `stroke-dashoffset` draw-in.
  - staggered bar-grow via CSS `transition` + `--stagger-step` index delays.
- **Reduced motion.** A single guard `prefers-reduced-motion: reduce` (matchMedia in JS + CSS media query) disables count-up, draw-in, sweep, and stagger uniformly — final values render instantly. This is the one global switch.
- **Restraint.** The hero scan-sweep is a slow, low-opacity recurring pulse on the same rhythm (not a flashy one-shot); telemetry rotation is slow and quiet.

## 5. The Lore System — cross-cutting, subtle

Lore is "opt-in to notice": visible if you look, ignorable if you don't.

- **Spell-card micro-captions.** Major section headers gain an optional faint secondary caption in low-opacity JetBrains Mono — a quiet spell-card-style label (e.g., a kanji/romaji subtitle). Small, dim (`opacity ~0.35`), never competing with the heading.
- **Keep existing framing.** "Scarlet Mist Barrier: Active" status badge stays — it is good, truthful lore.
- **No new loud effects.** Danmaku/mist/spirits remain confined to the existing opt-in fun-panel. We do not add new always-on canvas lore.
- Captions are data-free static text in the template; they carry no functional meaning and degrade trivially.

## 6. Components

### Component 1 — History pipeline (backend)

- **`merge.py::update_history(stats)`** — read `history.json` (recover to `[]` on missing/corrupt), append one snapshot, rotate to the last 90, write back.
  - Snapshot shape:
    ```json
    {"t": "<iso>", "total": int, "max_speed": int, "avg": float,
     "median": float, "p90": float,
     "vless": int, "vmess": int, "trojan": int, "ss": int, "hy2": int,
     "bs": int, "chs": int, "countries": int}
    ```
- **`build_html` deltas + series.** Compute deltas vs the previous snapshot and a compact series for sparklines. New template tokens (all swept by the existing safety-net if unfilled):
  - `{{TREND_NODES}}`, `{{TREND_SPEED}}` — signed percent strings (e.g. `+4.2`), or empty when <2 snapshots.
  - `{{HISTORY_JSON}}` — JSON of the last ~30 points reduced to the fields the sparklines need (`[{t,total,max_speed}]`), embedded as `data-history` on `#stats-data`.
- **`update.yml`** — add `history.json` to `PUBLISH_FILES`. Git is the storage; rotation bounds size.
- **Call order in `merge.py::main`:** aggregate stats → `update_history(stats)` → `build_html(...)` (so deltas use the just-appended point vs the prior one). `build_html` reads the rotated history for series/deltas.
- **Test:** `tests/test_history.py` — append, rotation cap at 90, recovery from a corrupt/missing file, delta correctness with <2 and ≥2 points.

### Component 2 — Hero "Live Command Center"

All hooks added to `template.html`; behavior in `main.js`; styling in `style.css`.

- **Count-up** on `#hero-max-speed` and `#hero-total-nodes` via `animateCount`, fed from the embedded `data-node-stats` (existing) — keeps `healPlaceholders()` as the value source of truth.
- **Trend chips** beside each number: `▲ +N%` (green) / `▼ −N%` (red) / hidden when no history. Values from `{{TREND_NODES}}` / `{{TREND_SPEED}}` (or derived client-side from `data-history`).
- **Sparkline** under the dashboard panel: thin inline SVG of node-count over the last ~30 snapshots, scarlet gradient stroke, draw-in on reveal. Hidden when <2 points.
- **Live sys-log:** extend `initLocalTime` to render relative time ("2 ч назад") on `[data-utc]`, plus a slow rotator cycling 3–4 **truthful** telemetry lines (median speed, survival %, top country) sourced from embedded stats.
- **CSS-only ambiance:** pulse on `.status-badge .dot`; a slow low-opacity scarlet scan-sweep over `.dashboard`. Both off under reduced-motion.

### Component 3 — Statistics showcase

- Animate existing protocol/class/country bar fills (width transition) with `--stagger-step`; count-up on `.chart-bar-value` and summary values.
- "Сводка по скорости": add Δ-vs-previous chips for max & median (reuse trend-chip component). Hidden without history.
- Country chart: count-up + a subtle one-pass shimmer on bar fills, on the shared rhythm.
- Everything gates on data presence (existing `|| '—'` fallback and `if (total===0) return` preserved).

### Component 4 — Subscriptions / onboarding polish (conservative)

- Unify copy feedback: every copy button shows an inline "Скопировано ✓" state with the same timing/style.
- "Freshness" indicator in the subscriptions modal: how long ago data updated (reuses relative-time helper).
- Smooth wizard step transitions onto the shared motion system.
- **Stretch / optional:** protocol/country filter chips in the subscriptions modal — to be scoped against the current modal markup during planning; not committed in this spec.

## 7. Data flow

```
main.py (per shard) ──► data/stats_*.json ──► (artifacts)
merge.py::main ──► aggregate stats
              ──► update_history(stats) ──► history.json (rotate 90)  ──► git
              ──► build_html(stats, history)
                     ├─ replace {{...}} incl. {{TREND_*}}, {{HISTORY_JSON}}
                     ├─ safety-net sweep ({{TOKEN}} → "—")
                     └─ write index.html  ──► git ──► Vercel deploy
browser: init() ──► healPlaceholders() ──► reveal conductor ──► animateCount / drawSparkline / bars
```

No runtime network calls; the page is self-contained. `/history.json` is published for power users/future use only.

## 8. Error handling & degradation

- **No / short history:** trend chips and sparkline hidden; charts still render current snapshot.
- **Corrupt history.json:** `update_history` recovers to `[]`; never throws into `main`.
- **Missing substitution:** existing server safety-net + `healPlaceholders()` keep raw tokens off-screen.
- **Reduced motion:** single global guard renders all final states instantly.
- **No new failure modes** introduced (no fetch, no external lib).

## 9. Testing

- **Python:** `tests/test_history.py` (rotation, recovery, deltas). Run existing suite to confirm no regressions in `merge.py`.
- **Build verification:** regenerate `index.html`; `node --check config/web/main.js`; assert `grep '{{' index.html` shows only in-JS guards, zero visible tokens.
- **Manual:** open `index.html` with a synthetic `data-history`; verify choreography order, count-up, sparkline draw, trend chips, reduced-motion off-state, mobile layout.

## 10. Build sequence

1. **Component 1** — history pipeline (unblocks trends). Ship + test first.
2. **Motion system + Lore system** — shared CSS/JS primitives (foundation for 2 & 3).
3. **Component 2** — hero live command center.
4. **Component 3** — stats showcase.
5. **Component 4** — subscriptions/onboarding polish.

## 11. Out of scope

- Backend node-validation logic (`go_core`, `core/`), already addressed separately.
- The deploy fix (`vercel.json`, placeholder self-heal) — done, only **preserved** here.
- Deep subscription filtering (Component 4 stretch) — deferred unless trivial.
- Any new external library, framework, or build step.
